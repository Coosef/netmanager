XI · Questions

Before the crossing.

How does device counting work?
Each managed network device — switch, router, firewall, AP, server NIC — counts as one. Virtual machines, clients and wireless endpoints are free. Sub-interfaces do not multiply the count.
Is on-premise installation possible?
Yes. Charon ships as a self-contained Docker Compose stack and a Kubernetes Helm chart. Air-gapped installations are supported on Enterprise tier with signed offline update bundles.
Where is the data stored?
On your own PostgreSQL and Redis instances. Charon never phones home. Cloud SaaS uses regional Frankfurt or Istanbul data centers with at-rest AES-256-GCM encryption.
Is multi-factor authentication required?
For admin and super-admin roles, yes — and it cannot be turned off. Operator and viewer accounts can be configured per-organization policy. Authenticator apps, email and hardware keys (FIDO2) are supported.
How does the first 30 days work?
A network engineer from our team co-pilots the rollout. Day 1: agent install, discovery, baseline. Week 2: playbook templates, SLA policies. Week 4: first incident dry-run together. No charge until you say yes.
What about agent overhead on small devices?
The proxy agent is ~12 MB resident memory and uses under 0.5 % CPU during idle polling on a Raspberry Pi 4. It does not need to run on the network device itself — it sits on any reachable host nearby.
Do you support white-labeling?
On Enterprise tier. Custom domain, brand colors, logo, login screen and email templates. The mythological dressing is optional.
What happens if my contract ends?
Your data, config, audit log and golden baselines export as portable JSON + plain config files. No lock-in, no proprietary format. The boat docks where you ask it to.
